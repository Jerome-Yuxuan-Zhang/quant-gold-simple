# Security Policy

## Supported versions

当前仓库仍处于早期阶段，默认只维护 `main` 分支。

## Reporting a vulnerability

如果你发现以下问题，请不要直接公开贴出真实敏感信息：

- API key 泄漏
- 文档里误写了敏感路径或凭证
- 下载流程可能把密钥写入日志
- 依赖存在已知高危漏洞

建议处理方式：

1. 不要提交包含密钥的 issue
2. 先本地撤销敏感内容
3. 用私密渠道通知维护者
4. 描述复现步骤，但不要暴露密钥本体

## Scope

本仓库最值得优先关注的安全面：

- `.env` 和环境变量加载
- 远程数据请求与缓存
- GitHub Actions 工作流
- 依赖供应链风险

## Non-goals

以下内容更偏“研究有效性”，不属于传统安全漏洞，但仍值得提 issue：

- 标签泄漏
- 样本外评估失真
- 回测与目标口径不一致
- 报告结论与代码流程不一致
