# 08 研究日志（Research Log）

## 2026-03-26 `v0.1` 初始化

### 本次目标

建立一个可以公开展示、可以本地复现、可以继续扩展的黄金量化研究起点版本。

### 本次假设

- 黄金下一交易日方向，可能对滞后后的宏观变量与风险变量存在弱可学习结构（Weak Learnable Structure）。
- 在日频（Daily Frequency）层面，逻辑回归（Logistic Regression）足够胜任第一版基线。

### 本次完成内容

- 初始化仓库结构
- 建立 Alpha Vantage 与 FRED 数据源抽象
- 建立缓存（Cache）与标准化（Normalization）流程
- 生成特征框架（Feature Frame）与建模框架（Model Frame）
- 加入逻辑回归分类基线与 Ridge 回归诊断基线
- 生成 EDA 报告与模型报告
- 补齐分文件 Markdown 文档

### 当前局限

- 还没有用真实 API key 完整跑远程模式
- 还没有处理宏观发布时间（Release Timing）
- 还没有处理修订值（Revisions / Vintage Data）
- 还没有进入正式回测（Backtesting）
- 还没有做 walk-forward 验证

### 当前研究卫生判断

当前版本更像“研究基线工程（Research Baseline Engineering）”，而不是“交易策略系统（Trading Strategy System）”。

这个定位是正确的，应该继续保持。

### 下一步建议

- 先配置真实 API key
- 先跑通 remote 模式
- 检查真实数据的缺失、发布日期与对齐问题
- 再进入更严格的时间序列建模与 ML 基线扩展

## 2026-03-26 真实数据远程流程验证

### 本次目标

验证真实 API 数据能否顺利进入 `v0.1` 流程，并检查当前基线是否具备继续研究的价值。

### 本次发现

- Alpha Vantage 黄金与白银数据已成功拉取并缓存。
- FRED 宏观与风险代理序列已成功接入。
- 特征层原先采用全局外连接（Outer Join）会导致真实可用样本几乎被清空。
- 已改为以黄金交易日历（Gold Trading Calendar）为主表，再对外生变量做左连接、滞后与有限前向填充。

### 本次真实数据结果

- `feature_rows = 5222`
- `model_rows = 3643`
- 测试集逻辑回归准确率（Test Accuracy）约为 `0.489`
- 测试集朴素方向基线（Naive Direction Baseline）准确率约为 `0.507`

### 当前解释

这说明：

- 真实数据链路已经具备可运行性
- 当前基线模型还没有优于朴素基线
- 现在最需要做的是研究诊断（Research Diagnosis），而不是盲目加更复杂模型

### 当前结论

`v0.1` 的工程骨架已经成立，但研究层面还处于“先理解数据和基线为何弱”的阶段。

这是正常且健康的研究进展。

## 2026-03-26 多目标与特征子集诊断

### 本次目标

判断到底是“目标期限（Target Horizon）选得不合适”，还是“特征集（Feature Set）混入了过多噪声”。

### 本次关键结果

- `1d / 3d / 5d` 三种目标都已纳入诊断。
- 在多期限比较中，`5d` 目标明显比 `1d` 更接近有效研究方向。
- 在 `5d` 目标下，`gold_only` 特征子集测试集准确率约为 `0.635`。
- 同期朴素方向基线约为 `0.529`。
- 这意味着 `gold_only` 相对朴素基线的准确率优势约为 `+0.106`。

### 当前解释

这个结果非常重要，因为它说明：

- 当前不是“所有基线都没用”
- 而是“目标定义”和“特征选择”对结果影响极大
- 对于当前这套日频黄金任务，纯价格行为特征（Price-derived Features）暂时比混合全部宏观变量更有效

### 当前判断

下一阶段最合理的主线应该改成：

- 以 `5d` 目标为重点诊断对象
- 以 `gold_only` 作为新的强基线（Strong Baseline）
- 再逐步检查哪些宏观变量能够在此基础上真正提供增量信息（Incremental Information）

## 2026-03-26 十年滚动样本外回测

### 本次目标

把当前最稳的机器学习主线接入一个最小可审计的回测原型，并用最近 10 年窗口做滚动样本外检验。

### 回测设置

- 目标：`5d` 方向
- 特征：`gold_only`
- 模型：逻辑回归（Logistic Regression）
- 回测窗口：最近 10 年
- 初始训练：3 年
- 再训练频率：21 个交易日
- 交易成本：5 bps
- 交易规则：预测上涨则次日做多，否则空仓

### 本次结果

- 策略总收益约为 `1.98`
- 买入持有总收益约为 `2.57`
- 策略年化收益约为 `11.36%`
- 买入持有年化收益约为 `13.34%`
- 策略夏普约为 `0.87`
- 买入持有夏普约为 `0.94`

### 当前解释

这个结果很关键：

- 机器学习信号不是完全无效，因为它能形成正收益路径
- 但它目前还没有优于简单买入持有
- 说明“预测有一点信息量”并不自动等于“交易上更优”

### 当前结论

回测层已经打通，但研究结论必须保持克制：

- 可以说“已经实现十年样本外回测原型”
- 不可以说“策略已证明优于基准”

## 2026-03-26 机器学习优化与防过拟合

### 本次目标

在当前 `5d + gold_only` 主线下，不是盲目选择训练集最强模型，而是选择“验证集更稳、过拟合更低”的模型。

### 优化原则

本次优化采用：

```text
objective_score = validation_accuracy - 0.6 * max(train_accuracy - validation_accuracy, 0)
```

也就是说：

- 验证集表现高更好
- 训练集和验证集差距太大要扣分
- 宁可保守，也不追求训练集漂亮数字

### 本次结果

当前被选中的最优模型是：

- `gradient_boosting`
- 参数：`n_estimators=100, learning_rate=0.02, max_depth=2`

它的表现大致为：

- 训练集准确率约 `0.578`
- 验证集准确率约 `0.566`
- 测试集准确率约 `0.561`
- 泛化落差约 `0.012`

### 最重要的研究发现

这个经过防过拟合选择的模型，在预测任务上更稳，但放进十年滚动样本外回测后，结果并没有优于此前的逻辑回归版本。

这说明：

- “预测更稳” 不等于 “交易更优”
- 机器学习模型选择和交易系统选择不是同一个问题
- 以后不能只看训练/验证/测试准确率，还要看样本外回测表现

### 当前结论

当前项目已经进入一个更成熟的研究阶段：

- 模型优化不能只看预测指标
- 回测不能替代模型选择
- 两者要并行看，并在冲突时做清楚的解释
